## 自作Webサイトのトレンド
### WordPress
+ ブログサービス自作で定番なのは WordPress。
+ WordPressは、Webサイトを構築する際に使用するソフトウェア。
+ [メリット]
  1. 無料で利用できる。
  2. 初心者でも簡単にWebサイトを作成できる。
  3. 用意されているテンプレートが多く、さまざまなタイプのサイトを作成できる。
  4. ブログやアフィリエイトに適している。
  5. 豊富なプラグインで拡張性が高い。
  6. 世界中に数多くの開発者・利用者がいる。
  7. 独自ドメインを利用できる。
  8. SEOに有利な作りになっている。
  
+ [デメリット]
  1. WordPress「だけ」でWebサイトができる訳ではない。ドメイン、サーバー設定が必要。
  2. 多く使用されているのでハッカーの標的になりやすい。
  3. 古いソフトなので表示速度が遅い。
  4. 定期アップデートが必要。プラグインもアップデート時の対応が必要になる。
  5. DBを使っているので、DB分のメンテナンス費用、サーバー利用料が掛かる。

### 静的サイトジェネレータ
+ WordPressのような動的サイトのデメリットを補うように出てきた。
+ コンテンツ（ヘッドレスCMSなど）を取り込み、選択したテンプレートを適用して、静的なHTMLページを生成する。
+ [メリット]
1. 静的で、DB等の動的利用がないため、セキュリティの心配がほぼない。
   1. 「Repositoryに記事データを全て格納」するのでDBバックアップが必要ない。
2. 静的なため、動作が速い。
3. 種類が多い。
   1. https://jamstack.org/generators/
4. サーバー構築が簡単。無料サービスが提供されている。

+ [デメリット]
1. プログラミング知識が必要。
2. 「お問合せフォーム」「ブログの投稿やコメント」「アクセス制限」等の動的な機能を付けれない。
3. ビルド処理によるタイムラグ。ビルドが必要なためコンテンツの編集からサイトへの反映までにタイムラグがあり、膨大なページ量のWebサイトには向いていない。

### Jamstack
+ 静的サイトジェネレータでの一般化しつつあるシステム構成。
+ J：JavaScript。コードを書くために使用されるプログラミング言語。
+ A：API。ウェブサイトのデータ（コンテンツ）にあたる部分。
+ M：Markup。フロント部分。
+ 但し現在はJavaScript、APIは必須ではなくなっている。
